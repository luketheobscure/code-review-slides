This is where AI falls down hardest
PDF story -> we need pds -> no we need integration -> llm can't find that.
Before you write a prompt, before you write a line of code — understand what you're actually trying to accomplish. Dig in with product. What problem are we solving? What does success look like? Then look at the full system. Is there existing work we can reuse? An existing service, function, or pattern? AI will happily build something that already exists, or something that contradicts how the rest of the system works. "The reviewer LLM still only sees the code. It doesn't see the specifications, it doesn't see the database, it doesn't know what scale we're working on, it doesn't know our expectations — some of which aren't written anywhere." AI reviews code. It does not review intent. That's your job.
