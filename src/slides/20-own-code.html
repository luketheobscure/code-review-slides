Own your code.
Understand your code.

  • 96% of devs distrust AI-generated code — only 48% review it before committing
  • 38% say reviewing AI code is harder than reviewing code a person wrote
  • "Verification debt": the understanding you used to earn by writing has to be rebuilt by reading

The Register, Jan 2026

Verification paradox: 96% distrust AI output but only 48% review it before commit. 38% find reviewing AI code harder than human-written code — you have less context for what the writer was thinking. Werner Vogels (AWS CTO) calls this "verification debt": writing code is how you build understanding of a system, and when a machine writes it, you have to rebuild that understanding through careful reading or you don't actually own the code. Toil doesn't go away — it shifts from authorship to verification. If you didn't write it, you have to understand it before you put your name on it.
