Invest in verification.

  • Review asks: does this code look right? Verification asks: does it behave correctly?
  • Generate tests from your acceptance criteria, let the requirements define the bar, not the implementation
  • Tools like Rovo Dev and Qodo check the PR against the linked ticket (or roll your own)
Review and verification are different things. Review asks: does this code look right? Verification asks: does it actually behave correctly — and does it do what was actually asked? One of the best uses of AI in testing: feed it your acceptance criteria and let it write the tests before the code ships. The requirements become the test suite. That's a fundamentally different bar than "does this diff look fine." Prefer integration and E2E tests — AI can game unit tests because it was trained on code that passes unit tests. Real systems, real data, real load — much harder to fool. Tools like Atlassian's Rovo Dev and Qodo now connect PRs directly to Jira tickets and validate the code against acceptance criteria. Atlassian uses this internally — they cut PR cycle time by 45%. Side benefit: catches scope creep when an AI agent "helpfully" changes things it wasn't asked to touch. Invest in verification infrastructure now, or pay interest forever.
