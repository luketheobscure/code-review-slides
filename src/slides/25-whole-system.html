Review the whole system.

  • Every PR is a window into the process that created it — not just the code
  • Ask: does this reveal that our agent instructions need updating? (agents.md, system prompts)
  • Ask: did the agent reinvent something that already exists? Update your shared skills and context
When an AI agent produces a PR, that PR is not just code. It's a window into the process that created it. The question isn't only "is this code correct?" It's: is the system that produced this code working as intended? Look for signals: did the agent go outside its scope? Did it rebuild something that already exists? Does this PR reveal that your agents.md or system prompts are incomplete or outdated? Update them. And treat the prompt like source code. Version-control it. Review prompt changes with the same rigor as code changes — they have the same blast radius. The full system to review: prompt → agent behavior → generated code → tests → deployment. Most teams are only looking at one layer.
